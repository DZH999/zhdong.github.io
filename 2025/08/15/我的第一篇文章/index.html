Hexo + GitHub Pages 个人博客搭建与笔记管理终极指南

一、核心概念

  1. Hexo：基于 Node.js 的静态博客框架，用 Markdown 写文章，生成 HTML 部署。

  2. GitHub Pages：免费托管静态网站的服务，自动关联 用户名.github.io 仓库。

  3. Markdown：轻量级标记语言，支持代码块、表格、图片等（文件扩展名 .md）。

四、常见问题解决方案

问题现象 原因 解决方案
样式/图片404 url 或 root 配置错误 检查 _config.yml 路径，确保末尾有 /

五、效率工具推荐

  1. 写作工具：

    • Typora（实时渲染 Markdown）

    • VS Code + Markdown All in One 插件

  2. 图床工具：

    • PicGo（自动上传图片生成链接）

    • 腾讯云 COS/Aliyun OSS（付费但稳定）

  3. SEO 优化：

六、高级技巧

  • 自动化部署：通过 GitHub Actions 实现提交代码自动部署。

  • CDN 加速：将静态资源托管到 jsDelivr。

  • 自定义域名：

    1. 在 source/CNAME 写入域名（如 blog.example.com）。

    2. DNS 解析添加 CNAME 记录指向 用户名.github.io。

七、示例命令速查表

功能 命令
新建文章 hexo new "标题"
本地运行 hexo server
生成静态文件 hexo generate（或 hexo g）
部署到 GitHub hexo deploy（或 hexo d）
清理缓存 hexo clean

八、总结

  1. 写作流程：
    hexo new → 编辑 Markdown → hexo server 调试 → hexo deploy 发布。

  2. 内容管理：

    • 文章在 source/_posts/

    • 图片随文章存放或集中放 source/images/。

  3. 持续维护：

    • 定期备份 source/ 文件夹。

    • 通过 Git 管理博客源码（如推送到私有仓库）。
